# Repository guidance

- Keep the public site visually restrained, concise, and free of public-voting flows.
- Do not invent award dates, fees, winners, category counts, judges, sponsors, or programme claims.
- Route public content through the typed content-source boundary instead of embedding CMS data in components.
- Use `ManagedPicture.astro` for programme images so local assets and future R2 URLs retain one rendering contract.
- Preserve keyboard access, visible focus, image alt text, and reduced-motion behavior.
- Run `npm run verify` and `npm run test:e2e` before release handoff.
